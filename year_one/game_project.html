The brief:

In the second half of our first term, we were assigned a group project of creating a game to educate children aged 12-15 on the Sustainable Development Goal 7 (SDG7). The group project pulled from members across several disciplines such as digital designers, 3D environment artists and CAD students. We each had a role to play within our group and I was tasked with creating the puzzles for the game, which would allow the player to progress upon solving them.

Researching and brainstorming:

Our game had several different distinct areas that would be accessed via portals. In each area there was a different biome which utilised various forms of cleaner energy. The initial idea was to have each area have a unique puzzle style. Due to us not having time to make all 5 areas we planned, we settled on the desert area. When I was tasked with making the puzzles, I decided straight away that a numberlink puzzle would best fit the area, as the energy source for the area was solar powered. The numberlink simulated a circuit that would cause the solar panels to work again.

Prototyping:

My main inspiration I drew upon while making the prototypes and eventual final version of my game was another well known numberlink puzzle by the name of Flow FreeÂ®. The software I used to make the game was Unity and Sublime Text, written in C#. My game went through several distinct phases, with a roadmap of goals that defined key stages in the development. The key stages are as follows:

  • Create a grid of cells that can be toggled on and off, generated from a single cell.
  • Drag through the cells to toggle them on and off, but only if you are dragging from an already active cell or the start point.
  • Only be able to drag from the end cell of the path, and enabling erasure of the line by doubling back or overlapping the path with itself.
  • Multiple colours (and preventing paths of different colours overlapping)


With this path it allowed me to create a fully functional numberlink puzzle which is fully rendered using code within Unity, as opposed to working with sprites to create the lines and end points.

The final outcome:

The final outcome of my work was a numberlink which was rendered entirely using C# and Unity, would allow for easy editing using the level maker script and had multiple directions it could be taken to expand upon the idea for making more challenging puzzles for the specific area, which could unlock collectables.
